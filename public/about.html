How It Works

Resume Forge analyzes job descriptions and rewrites your resume to maximize ATS compatibility, keyword matching, and relevance — all while keeping your claims truthful.

1
Getting Started
Set up your resume data and configure AI providers
1
Create Your Resume Data File
Copy resumeData.example.json to resumeData.json and fill in your real details — name, skills, experience, projects. This is your single source of truth for all resume generations.
2
Configure Environment Variables
Copy .env.example to .env and fill in your AI provider credentials (AWS Bedrock keys or Gemini API key), SMTP settings for email sending, and output directory preferences.
3
Install Dependencies
Run npm install and optionally install LibreOffice for PDF conversion. Without LibreOffice, resumes are generated as DOCX files.
4
Start the Server
Run node server.js to start the web UI. On startup, it validates your resumeData.json and shows any issues on the Generate page.

AI Prompt to Generate resumeData.json

If you don't want to write it manually, copy this prompt into any AI chat and it will generate the file for you:

Generate a resumeData.json file for my resume with the exact structure shown below. Fill it with my real details: Name: [Your Name] Email: [Your Email] Role: [e.g., Full Stack Developer] Experience: [X years], started [Month Year] Stack: [e.g., MERN, Python/Django, Java/Spring] Current company: [Company Name] Projects I worked on: [List each project with 2-3 sentences about what you built] Personal projects: [List each with a brief description] Skills: [List all your technical skills] Education: [Your degrees] Use the structure from resumeData.example.json — keep all section names, field names, and nesting exactly the same. Write detailed project descriptions (100+ words each) covering what was built, your role, and technologies used.Click to copy
2
Generation Pipeline
What happens when you click Generate
1
Pre-flight Checks
Verifies AI provider credentials, checks that resumeData.json and template.docx exist, and tests LibreOffice availability for PDF conversion.
2
JD Validation
Validates the pasted job description has enough content to work with (minimum 100 characters).
3
AI Analysis + Rewriting
Sends your resume data and the JD to the AI provider. The AI extracts keywords, matches them against your skills, identifies gaps, and rewrites your resume title, summary, bullets, skills, and project descriptions to maximize keyword alignment. It also generates application emails and LinkedIn DMs.
4
ATS Scoring
Runs a deterministic scoring algorithm: keyword match percentage, title match, experience gap penalties, and hard-reject gates (e.g., if the JD requires Java but your stack is JavaScript). Scores above 70% are good, 50-69% are mid-range, below 50% need attention.
5
Document Generation
Creates a formatted DOCX resume from your template, then converts it to PDF via LibreOffice. The output is saved to your configured directory and logged in history.

Skill Claim Rules

  • Exact Match — JD keyword exists in your skills or project tech. These are directly used.
  • Can Claim — JD says a generic term (e.g., "JavaScript frameworks") and you have a specific one (e.g., React). Only for generic-to-specific matches.
  • Cannot Claim — Skills listed in meta.cannotClaim plus any tech that's a different tool in the same category (has MongoDB, can't claim PostgreSQL from JD).
  • Missing — Critical JD requirements you don't have. Shown in the skill gaps report.

Resume Data Structure
How to structure your resumeData.json file

Your resumeData.json is the single source of truth. The AI never invents skills or experience — it only rewrites what's already in this file. The more detailed your project descriptions are, the better the AI can match JD keywords.

Top-Level Sections

  • personalInfo required — Name, email, phone, links. Name and email are required, rest optional.
  • meta required — Experience start date (e.g., "Jun 2022"), stack, cloud provider, project lists, and cannotClaim array for skills you don't have.
  • professionalSummary required — A default summary string (200+ chars) that gets rewritten per JD. Optional keywords array.
  • skills required — Split into frontend, backend (required), and toolsDevOps, other (optional). Each is an array of strings.
  • experience required — Array of jobs. Each needs company, title, duration. Can have nested projects array and bullets array.
  • projects required — Personal/freelance projects. Each needs name and description. Optional coreTech and stackUsed.
  • education optional — Array with degree and institution required.

Key Fields Explained

  • meta.experienceStart — Used to calculate total years of experience. Format: "Mon YYYY" (e.g., "Aug 2023").
  • meta.coreProjects — Work project names that always get highlighted in bullets. Must match names in experience[].projects[].name.
  • meta.cannotClaim — Skills the AI must never claim you have, even if a JD asks for them. Prevents false claims.
  • experience[].projects[].coreTech — Key technical achievements per project. Used for keyword matching against JDs.
  • experience[].projects[].description — Detailed description (100+ words recommended). The AI rewrites from this, so more detail = better output.

Minimal Schema

{
  "personalInfo": {
    "name": "Your Name",          // required
    "email": "you@email.com"      // required
  },
  "meta": {
    "experienceStart": "Jun 2022", // required, "Mon YYYY"
    "stack": "MERN",               // required
    "cannotClaim": ["Java", "Go"]  // skills you don't have
  },
  "professionalSummary": {
    "default": "Your professional summary (200+ chars)..."
  },
  "skills": {
    "frontend": ["React", "TypeScript"],
    "backend": ["Node.js", "MongoDB"]
  },
  "experience": [{
    "company": "Company Name",
    "title": "Your Title",
    "duration": "Jun 2022 - Present",
    "projects": [{
      "name": "Project Name",
      "description": "Detailed description of what you built...",
      "coreTech": ["Key Tech 1", "Key Tech 2"]
    }]
  }],
  "projects": [{
    "name": "Side Project",
    "description": "What it does and how you built it..."
  }]
}

AI Providers
Supported AI backends and how to configure them

AWS Bedrock (Default)

  • Set AI_PROVIDER=bedrock in .env
  • Requires AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY
  • Models: Claude Haiku (default), DeepSeek, Qwen, GLM — selectable from the dropdown
  • Costs are tracked per-call and logged to logs/bedrock_costs.json

Google Gemini

  • Set AI_PROVIDER=gemini in .env
  • Requires GEMINI_API_KEY
  • Auto-rotates through model pool on rate limits (Gemini Pro, Flash, etc.)
  • Free tier available with rate limits

Adding a Custom Provider

  • Create a new file in services/providers/ extending BaseProvider
  • Implement tailorResume(jd, resumeData) method
  • Register it in services/providers/registry.js
  • See services/providers/README.md for the full guide

CLI & Claude Code Usage
Using the tool from terminal or with Claude Code

Web UI

Run node server.js and open http://localhost:5003. Paste a JD, pick your model, and click Generate.

CLI via index.js

Run node index.js to use the command-line interface. It reads the JD from your system clipboard (xclip on Linux), runs the same pipeline, and outputs the resume to your configured directory.

Claude Code Integration

If you use Claude Code (Anthropic's CLI), say "tailor resume" and it will follow the workflow defined in CLAUDE.md: read clipboard for JD, analyze keywords, rewrite your resume, run the generation script, and display results — all in one go.

7
Resume Validation
Automatic structure checking on startup

When the server starts and when you open the Generate page, your resumeData.json is automatically validated against the expected schema. You'll see a banner at the top of the Generate page showing:

  • Green — All good. Shows your name, skill count, and project counts.
  • Amber — Valid but has warnings (e.g., short descriptions, missing optional fields).
  • Red — Has errors that need fixing before you can generate (e.g., missing required sections or fields).

If resumeData.json doesn't exist at all, the banner expands automatically with setup instructions and an AI prompt to help you create the file.

You can also check validation programmatically at GET /api/validate-resume.
